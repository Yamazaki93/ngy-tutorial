Getting Started

Add the library by using npm with npm install ngy-tutorial or yarn with yarn add ngy-tutorial

  1. Add the module to your root module with NgyTutorialModule.forRoot() under imports in your app.module.ts.
  2. Place element <lib-ngy-tutorial></lib-ngy-tutorial> in your app.
  3. You can now inject and use NgyTutorialService service to control the tutorial, see the API for available commands and options.

API

Module - NgyTutorialModule

forRoot(options: IInitializationOptions)

Initializes the module with the passed in options, see IInitializationOptions for available options

Service - NgyTutorialService

addStep(title: string, prompt: string, options?: IStepOption)

Adds a tutorial step to the entire tutorial. title is the one-line heading on the top of the dialog and prompt is the detailed texts displayed below that. If the tutorial has started, the newly added steps will only appear on the next run. options? is the optional options for this step, see IStepOption for available options

removeStep(stepNumber: number)

Removes a tutorial step with the given step number. stepNumber is the zero-indexed number specifying the index of the step to remove. If the tutorial has started, the removal will take effect during the next run.

showTutorial()

Starts the tutorial sequence. If tutorial is already started, this will restart the tutorial.

hideTutorisl()

Stops the tutorial sequence. If the tutorial has not started, this has no effect.

nextStep()

Advances the tutorial to the next step. If the tutorial is at the last step, calling nextStep will end the tutorial.

previousStep()

Reverts the tutorial to the previous step. If the tutorial is at the first step, calling previousStep will end the tutorial.

Interface - IInitializationOptions

Field classToAdd?: IStepClasses

Specifies classes to apply to the prompt dialog to allow customized styling. See IStepClasses for list of available sub-fields

Field actionTexts?: IActionTexts

Specifies the button texts that appear on the prompt. See IActionTexts for list of available sub-fields
