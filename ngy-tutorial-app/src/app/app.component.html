Getting Started

Add the library by using npm with npm install ngy-tutorial or yarn with yarn add ngy-tutorial

  1. Add the module to your root module with NgyTutorialModule.forRoot() under imports in your app.module.ts.
  2. Place element <lib-ngy-tutorial></lib-ngy-tutorial> in your app.
  3. Inject and use NgyTutorialService.addStep(...) to add steps to your tutorial
  4. Call NgyTutorialService.showTutorial() in ngOnInit to start the tutorial.

API

Module - NgyTutorialModule

forRoot(options?: IInitializationOptions)

Initializes the module with the passed in options (optional), see IInitializationOptions for available options

Service - NgyTutorialService

Field - tutorialInProgress: Observable<boolean>

Contains the current tutorial state. true if the tutorial is in progress, fasle otherwise

Field - CurrentStep: Observable<number>

The current tutorial step as a zero-indexed number. NOTE, this should not be used to determine if tutorial is in progress, use tutorialInProgress instead.

addStep(title: string, prompt: string, options?: IStepOption)

Adds a tutorial step to the entire tutorial. title is the one-line heading on the top of the dialog and prompt is the detailed texts displayed below that. If the tutorial has started, the newly added steps will only appear on the next run. options? is the optional options for this step, see IStepOption for available options

removeStep(stepNumber: number)

Removes a tutorial step with the given step number. stepNumber is the zero-indexed number specifying the index of the step to remove. If the tutorial has started, the removal will take effect during the next run.

showTutorial()

Starts the tutorial sequence. If tutorial is already started, this will restart the tutorial.

hideTutorisl()

Stops the tutorial sequence. If the tutorial has not started, this has no effect.

nextStep()

Advances the tutorial to the next step. If the tutorial is at the last step, calling nextStep will end the tutorial.

previousStep()

Reverts the tutorial to the previous step. If the tutorial is at the first step, calling previousStep will end the tutorial.

Interface - IInitializationOptions

Optional Field classToAdd?: IStepClasses

Specifies classes to apply to the prompt dialog to allow customized styling. See IStepClasses for list of available sub-fields

Optional Field actionTexts?: IActionTexts

Specifies the button texts that appear on the prompt. See IActionTexts for list of available sub-fields

Interface - IStepOption

Optional Field placement?: StepPromptPlacement

Specifies the placement of the prompt for each step. This will only have effect if relatedElement is set properly. The prompt is placed at the center of the screen by default.

Optional Field relatedElement?: string

The ID of the DOM element that this step is focusing on. Cover will be applied to the screen and show a "viewing window" highlighting the element of interest. If the ID is not found on the DOM, the behavior is the same as if this option is not set.

Optional Field disableInteraction?: boolean

If this is set to true. Interaction with components other than the tutorial is disabled, the "viewing window" will still focus on the element provided in relatedElement but no action can be perfored. It cannot be set to true if disableNext is set to true.

Optional Field disableNext?: boolean

If this is set to true, the next button (the finish button, if it is at last step) is disabled. To progress forward, you can still call NgyTutorialService.nextStep(). It cannot be set to true if disableInteraction is set to true.

Optional Field disablePrevious?: boolean

If this is set to true, the previous button (the skip button, if it is at first step) is disabled. To progress backward, you can still call NgyTutorialService.previousStep(). NOTE: It CAN be set to true when disableNext is set to true or disableInteraction is true, it is up to your app to determine the availability of "previous" step.

Optional Field classToAdd?: IStepClasses

Specify classes to add to the prompt dialog for this class. The classes will be applied on top of the classes specifies in the initialization options. Except for classes for various *Icon, they will override the classes specified in the initialization options.

Optional Field actionTexts?: IActionTexts

Overrides the text for the action buttons in the prompt for this step.

Interface - IStepClasses

Optional Field container?: string[]

Specifies a list of class names to apply to the prompt container

Optional Field title?: string[]

Specifies a list of class names to apply to the prompt title

Optional Field prompt?: string[]

Specifies a list of class names to apply to the prompt text

Optional Field previousBtn?: string[]

Specifies a list of class names to apply to the "Previous Step" button

Optional Field previousIcon?: string

Specifies the class name for the icon of the "Previous Step" button. If no classes are supplied by both IStepOption and IInitializationOptions, the icon is hidden

Optional Field nextBtn?: string[]

Specifies a list of class names to apply to the "Next Step" button

Optional Field nextIcon?: string

Specifies the class name for the icon of the "Next Step" button. If no classes are supplied by both IStepOption and IInitializationOptions, the icon is hidden

Optional Field finishBtn?: string[]

Specifies a list of class names to apply to the "Finish" button. This will only have effect for the last step

Optional Field finishIcon?: string

Specifies the class name for the icon of the "Finish" button. If no classes are supplied by both IStepOption and IInitializationOptions, the icon is hidden

Optional Field skipBtn?: string[]

Specifies a list of class names to apply to the "Skip" button. This will only have effect for the first step

Optional Field skipIcon?: string

Specifies the class name for the icon of the "Skip" button. If no classes are supplied by both IStepOption and IInitializationOptions, the icon is hidden

Interface - IActionTexts

Optional Field next?: string

Specify the text for the "Next" button, the default is "Next"

Optional Field previous?: string[]

Specify the text for the "Previous" button, the default is "Previous"

Optional Field skip?: string[]

Specify the text for the "Skip" button, the default is "Skip"

Optional Field finish?: string[]

Specify the text for the "Finish" button, the default is "Finish"

Enum - StepPromptPlacement

This is used to specify the placement of the prompt during the tutorial. Specifying the placement only has effect if relatedElement is set properly. The prompt placement will be adjusted so that the prompt is always in view. Note, this means placing prompt improperly can cover the viewing window.

Above

The prompt is placed above the viewing window

Below

The prompt is placed below the viewing window

Left

The prompt is placed at the left of the viewing window

Right

The prompt is placed at the right of the viewing window
